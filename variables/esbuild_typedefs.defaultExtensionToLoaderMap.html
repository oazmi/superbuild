    Variable defaultExtensionToLoaderMapConst

    defaultExtensionToLoaderMap: Record<string, EsbuildLoaderTypeOrEmpty> = ...

    this dictates esbuild's default/native file-extension to loader mapping. it is roughly based on the following esbuild function: "/internal/bundler/bundler.go":DefaultExtensionToLoaderMap

    Note

    one important distinction to note down is the fact that the loader for the "no extension" case ("" empty string) is undefined, meaning that it will use esbuild's own default loader for it (which happens to be "js" for such cases).

    the reason why we don't directly assign a "js" loader to this case is because our super-build's transformer hooks may utilize the knowledge of whether a "default" was originally passed, or if an actual "js" loader was intended to be used by the prior onLoad hook's callback.
